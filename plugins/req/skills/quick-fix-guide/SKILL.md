# 快速修复助手

快速分析问题，生成简洁可执行的实现方案。

## 分析流程

1. **理解问题** - 错误信息、现象 vs 根因、影响范围
2. **定位代码** - 报错堆栈、功能模块、关键字搜索
3. **分析实现** - 当前逻辑、问题环节、最小改动
4. **生成方案** - 文件、改动、验证方式

## 方案评估

| 检查项 | 快速修复标准 | 超出时 |
|-------|------------|-------|
| 文件数量 | ≤5 个 | 升级为正式需求 |
| 代码行数 | ≤100 行 | 考虑拆分 |
| 数据库变更 | 无 | 升级为正式需求 |
| 新增API | 无 | 升级为正式需求 |
| 影响范围 | 单一功能 | 评估风险 |

## 输出格式

```
🔍 问题分析
[1-2句话说明问题根因]

💡 实现方案
1. 文件：path/to/file.go
   改动：[具体改动内容]

📏 改动量：小/中（约 XX 行）

✅ 验证方式
[如何验证修复成功]
```

## 完成标准

- [ ] 问题已解决/功能已实现
- [ ] 代码符合项目规范
- [ ] 简单自测通过

**原则：快速、简洁、聚焦**
