# 代码影响分析器

分析需求变更或代码修改对系统的影响范围。

## 分析维度

### 1. 直接影响
- 需要修改的文件列表
- 代码行数估算（新增/修改/删除）

### 2. 间接影响
- 调用链上下游
- 被调用方
- 共享数据影响

### 3. 接口兼容性
- 参数变更（新增/修改/删除）
- 向后兼容性评估
- 前端影响

### 4. 数据迁移
- 表结构变更
- 数据迁移需求
- 回滚方案

### 5. 风险评估
- 风险等级（低/中/高）
- 风险点列表
- 缓解措施

### 6. 测试建议
- 必测项
- 回归测试范围
- 性能测试需求

## 分析原则

1. **全面性**：覆盖所有可能的影响点
2. **准确性**：基于代码实际调用关系
3. **可操作性**：给出具体的应对措施
4. **风险导向**：重点关注高风险变更
