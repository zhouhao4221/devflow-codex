# PR 评审

> 本文件由 `/req:review-pr` 读取，注入项目特有的 PR 审查关注点。
> 删除不需要的章节即可——缺失时按通用代码审查维度执行。

## 什么时候用

> 哪些 PR 需要按本规范审查。

<!--
示例：
- 所有合并到主干 / develop 的 PR
- 紧急 hotfix 可降低部分非阻塞项
-->

## 必备输入

> 审查前需掌握的上下文。

<!--
示例：
- 关联需求 / issue 编号与验收标准
- 变更范围与影响面
- 项目编码规范、安全红线
-->

## 触发方式

<!--
示例：
- `/req:review-pr <PR编号>`
-->

## 优质输出标准

> 一次好的审查应覆盖的维度。

<!--
示例：
- 正确性 / 边界 / 错误处理
- 安全（注入、鉴权、敏感信息）
- 分层与复用是否符合 architecture.md
- 测试是否覆盖新增逻辑
-->
