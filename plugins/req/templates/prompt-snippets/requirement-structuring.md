# 需求结构化

> 本文件由 `/req:new`、`/req:edit` 读取，注入项目特有的需求拆解与表达约定。
> 删除不需要的章节即可——缺失时按需求模板通用结构组织。
>
> 注意：`requirement-analyzer` 助手受 Memory 隔离约束，本文件**只影响内容侧引导**，不得改变需求模板的章节结构与格式。
